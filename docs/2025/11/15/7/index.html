微信小程序开发技巧与踩坑记录

最近在写一个小程序项目，从零开始踩了不少坑，记录一下学习过程中遇到的技巧和问题。

环境与工具

开发工具用的是微信开发者工具，真机调试比模拟器靠谱太多。模拟器上正常的东西真机上不一定能用，特别是涉及硬件 API 的时候。

核心技巧

1. 彻底放弃 Web 思维

小程序不是浏览器环境，没有 window、document、localStorage 这些东西。一开始总是下意识写 document.querySelector，然后报错才反应过来。

  • 页面结构只能用小程序原生标签：<view>、<text>、<image>、<button> 等
  • 数据绑定用 this.setData({})，不能直接赋值 this.data.xxx = yyy，视图不会更新
  • 本地存储用 wx.setStorageSync 和 wx.getStorageSync

3. 列表渲染的性能优化

列表渲染用 wx:for，wx:key 是一定要加的，否则数据变更时小程序做全量 diff，性能很差：

长列表建议用虚拟列表的思路，只渲染可视区域的数据。我用了一个简单的分页滚动方案，onReachBottom 触底加载更多。

4. 条件渲染 wx:if vs hidden

wx:if 是真正的条件渲染，不满足条件时不渲染 DOM；hidden 只是 display:none。

  • 切换频繁用 hidden（省去创建销毁开销）
  • 初始条件大概率不满足用 wx:if（省内存）

5. 组件化开发

小程序支持 Component({}) 构造函数，生命周期和 Page({}) 不一样，别混用：

类型 生命周期
Page onLoad → onShow → onReady
Component created → attached → ready → detached

组件间通信用 this.triggerEvent('eventName', { data })，父组件通过 bind:eventName 监听。

坑2：图片路径问题

小程序包有大小限制（主包 2MB），所有图片尽量用网络图片或者放到云存储。本地图片路径在真机上有时会莫名其妙 404。

总结

小程序开发入门不难，但细节很多。最大的感触是必须忘掉浏览器那一套，遵循小程序的组件化思维。遇到问题先看微信官方文档，比搜博客管用。
